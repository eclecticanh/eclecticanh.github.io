Mobile Application Project

Project Documentation

Summary

For our Mobile Application Development class we created a web application using jQuery Mobile. For the first module we created a mock-up design for the website. The next week we added in a header, logo, content, and a footer section.

For week 3 we added in new pages with different lists of favorite things along with a simple form. Week four we added in widget, grids, and panels. Then in week 5 we created a theme using ThemeRoller and attached it to our site. Using jQuery Mobile was an interesting experience. It comes with numerous benefits for sizing to different sized screens. I however believe that this framework loads slowly, and unlike other HTML5 frameworks like Bootstrap,jQuery Mobile is no longer receiving updates and support. ThemeRoller was very simple and efficient to use, and was my favorite feature to use.

Website Structure

The website structure changed quiet a bit throughout the term. The final structure included a nav bar at the top that included a form labeled "Contact Us" and a widget that held links to three official websites.

The body of the application had a header, content, and footer section. In the body there are three links to different sections of the web page. Each section has it's own header, content, and footer section. The content of each is a different type of list. The final version applied different swatch theme combinations to each of these sections.

Buttons

The final website contains numerous links and buttons.

The panel at the top is clickable, and includes href links within it. Each clickable section within the body has a back button, and the forms contain submit buttons that commit values to JS variables.
